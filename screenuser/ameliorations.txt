AUDIT + AMELIORATIONS APPLIQUEES
Date: 2026-02-17

Contexte
- Site vitrine snack/pizza sur GitHub Pages
- Objectif principal: appels telephoniques + visibilite locale
- Pas de commande/paiement en ligne

1) UX + accessibilite (applique)
- Contraste augmente:
  - texte principal plus clair
  - texte secondaire remonte pour mieux respecter WCAG sur fond sombre
- Hierarchie typo clarifiee:
  - body en base plus lisible
  - titres plus nets
  - classes utilitaires text-body / text-body-muted
- Barre sticky de commande:
  - boutons de telephone centres visuellement
  - labels FR/EN selon la langue active
- Promo retravaillee:
  - badge + encart plus contrastes
  - fond distinctif
  - animation legere (pulse) sans perte de lisibilite

2) Images + performances (applique)
- WebP ajoute pour hero + logo:
  - src/assets/hero-bg.webp
  - src/assets/logo_pizzatiq.webp
- Lazy loading deja conserve sur les images menu
- Alt texts detailles sur les images menu (nom + description + categorie)
- Compression image cote admin:
  - upload converti en WebP (qualite ~0.82)
  - redimensionnement max largeur 1600 px avant envoi Supabase

3) Livraison + conversion (applique)
- Statut dynamique Ouvert/Ferme (heure Europe/Paris) sur accueil + contact
- Carte interactive (OpenStreetMap + Leaflet):
  - affichage rayon 20 km
  - point restaurant + point adresse client
- Verificateur d'adresse:
  - geocodage adresse
  - calcul distance
  - eligibilite livraison oui/non
  - estimation frais + minimum commande selon distance

4) SEO technique + structure (applique)
- Hreflang FR/EN dynamiques par page
- Canonical + OG/Twitter conserves et renforces
- Sitemap enrichi:
  - routes FR
  - routes EN
  - section actualites/news
- Vite minification explicite:
  - JS minify esbuild
  - cssMinify active
- H1/H2/H3:
  - un H1 par page principale
  - hierarchie de sections clarifiee

5) Contenu SEO (applique)
- Version anglaise:
  - /en

6) Legal + RGPD (applique)
- Mentions legales completees avec sections obligatoires:
  - editeur
  - hebergement
  - propriete intellectuelle
  - donnees personnelles/RGPD
  - cookies/analytics
- Formulaire contact RGPD:
  - consentement explicite obligatoire
  - minimisation des donnees
  - aucun stockage auto en base
  - envoi via client email (mailto)

7) Google Analytics - ce qui est fait dans le code
- Tag GA4 present dans index.html
- Page views SPA envoyees a chaque changement de route
- Events clics explicites:
  - call_click
  - menu_tab_click
  - home_menu_cta_click
  - home_contact_cta_click
  - website_click
  - language_switch_click
  - contact_form_mailto_click
- Stockage parallele des events dans analytics_events (dashboard admin)

8) Google Analytics - actions a faire dans l'interface GA4
- Admin > Data Streams > verifier stream Web actif (G-66NXKHP23T)
- Admin > Events:
  - verifier remontee des events ci-dessus
- Admin > Key events:
  - marquer en conversion:
    - call_click (priorite 1)
    - home_menu_cta_click (priorite 2)
    - contact_form_mailto_click (priorite 3, option)
- Reports/Explore:
  - rapport \"Intentions de commande\":
    - dimensions: eventName, pagePath
    - metriques: eventCount, users
  - rapport \"Parcours mobile\":
    - dimensions: deviceCategory, pagePath
    - metriques: views, keyEvents

9) Limite importante GitHub Pages (a connaitre)
- En HTTP brut:
  - /menu, /contact, /en/menu peuvent retourner 404 serveur
- En navigateur:
  - la route est restauree ensuite via fallback SPA
- Impact:
  - certains bots sans JS peuvent mal interpreter les routes internes
- Solution ideale long terme:
  - hebergement avec rewrites serveur 200 (Netlify / Vercel / Cloudflare Pages)
